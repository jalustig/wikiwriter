## Your Task

For each candidate section, decide:
1. Whether it needs editing (high-scoring sections ≥ 8.0 should be excluded as "quality sufficient")
2. Which edit modes to assign (one or more)

### Available Edit Modes

- **"Citation Repair"** — fix dead links, replace weak sources, correct unsupporting citations
- **"Claim Attribution"** — add inline citations to uncited factual assertions
- **"Section Expansion"** — develop thin or stub sections with room to grow
- **"Section Rewrite"** — rewrite biased, unclear, or outdated prose
- **"Synthesis Pass"** — improve global flow and concision (apply to whole article, not individual sections)

### Mode Assignment Routing

Use these signals to guide mode assignment:
- Low `citation_coverage` score → suggest **"Claim Attribution"** for affected sections
- Low `citation_quality` score → suggest **"Citation Repair"** for affected sections
- Low `npov` score → suggest **"Section Rewrite"** for affected sections
- Low `prose_quality` score → suggest **"Section Rewrite"** for affected sections
- Low `structural_completeness` for a section → suggest **"Section Expansion"**
- Multiple failing dimensions → assign multiple modes to the same section

Scores below 6.0 are "low" and should trigger the corresponding mode. Scores between 6.0 and 7.9 may warrant a mode if the section is important.

### Exclusion Criteria

Exclude a section (do NOT assign edit modes) if:
- Section score ≥ 8.0 → reason: "quality sufficient"
- The section is a non-prose navigation section (e.g., "See also", "External links", "References", "Notes", "Further reading") → reason: "non-prose section"

## Output Format

Return ONLY a JSON object with this exact schema — no preamble, no markdown fences:

{{
  "sections_to_edit": [
    {{"name": "History", "modes": ["Citation Repair", "Claim Attribution"], "rationale": "..."}}
  ],
  "sections_excluded": ["Lead", "See also"],
  "exclusion_reasons": {{"Lead": "quality sufficient", "See also": "non-prose section"}},
  "narrative": "2-3 sentence summary of the overall edit plan and the most important improvements to make."
}}

Every candidate section must appear in either `sections_to_edit` or `sections_excluded` — no section may be omitted.
